Contributing

If you want to contribute to one of my repositories by making an issue or pull request, please read through the following.

Issues

  • Issues can be in any format you'd like as long as they are readable and in English.
  • Avoid making duplicate issues. Check to make sure the issue you want to report hasn't been reported already.
  • Verify that something in the repository you are creating the issue in is directly responsible for your issue, and provide proof of this if possible.
  • Also verify that you have correctly installed and/or configured the item that is causing the issue.

Pull Requests

The following is a list of things I recommend you do when creating a pull request. You don't have to follow all of them. In the end all I care about is code that is easily readable and fits well with the rest of the code.

  • Try to keep your pull requests small and directed towards a single change. If you want to change or add multiple unrelated things, make separate pull requests for each of them.
  • Complete rewrites of large parts and extremely small changes probably won't be accepted unless you can provide proof that the change greatly improves performance and/or functionality.
  • Place comments next to code that I or other contributors may find confusing.
  • Use size 4 tabs instead of spaces.
  • Place global variables, includes, and usings at the very top of the file if possible.
  • Insert spaces between parenthesis ( ), angle brackets { }, and arguments as well as after commas. Leave square brackets [ ] and arrows < > without spaces.
  • Leave whitespace in between blocks of code that span multiple lines.
